Why?

You might ask "Why another language?".

I, of all people, would use an existing language for what I'm trying to do if possible.

So far I've tried several dozens of languages and none come even close to all of the following goals:

Goals

  1. Exploratory Programming: get results as soon as you ask, the "ask questions, get answers" property, a read-eval-print loop.
  2. Higher-order Programming: if I have to write another "for" loop, I'm going to scream.
  3. Numeric Tower: if using numbers, have them behave as in ordinary mathematics (the one everyone on the planet learns in high school) as far as possible.
  4. Reflectiveness: Allow finding out of what exists by asking the system.
  5. Simplicity: as simple as possible but no simpler (this means that there are few basic parts where each part is simple).
  6. Referential Transparency: Replacing a symbol by its definition may not change the meaning of the program.

(Haskell misses [1 3 4], it's closer than any other language)

Non-Goals

  • Low-level Systems Programming: I don't care about word size, manual memory management, memory mapped I/O, latches, Memory Barriers, ... . Neither does the language.
  • Speed: No, this language doesn't need to be faster than C.

What?

So far, there exist:

  • the 5D Language Specification
  • an interpreter for the language, see the 5D User Manual for how to run the interpreter.
  • documentation:
    For how to use the language, see 5D Tutorial and 5D Programming Manual.
    For how to use the standard libraries, see 5D Library Documentation.
