Syntax Elements

Names can contain every printable character except space and # and @, although operator names consist of different characters than other names.

Operator names consist of funny characters or math operator characters, not alphanumeric letters. Custom operators have to be registered, giving them a precedence level, before they can be used.

Some are preregistered: _ . ^ ** ⨯ * ⋅ / % : ++ $ + - = /= < <= ≤ > >= ≥ && || , ; | =>

Monoms after a name are function arguments, like everywhere in mathematics: sin k⋅x=sin(k⋅x) and not (sin k)*x.

On the other hand, sin k⋅x+φ=(sin k⋅x)+φ

Likewise, 5⋅f x=(5⋅f) x and not 5⋅(f x)

Mathematics

The usual calculator mathematics work:

5 + 2 => 7

5 + 2⋅3 => 11

Operator precedence is as you would expect, with one strange quirk: 2⋅f x means (2⋅f) x

5 + 2⋅3 => 11

define f \x x⋅x
5 + 2⋅(f 3) => 23
5 + 2⋅f 3 => ??????

Here, the function f is defined to be a function of x, giving the square of x.

Then the function f is called with the argument 3. The following works, too (although it has more parens for no reason):

define f\x x⋅x
5+2⋅(f(3)) => 23

Or if it's the first operand of multiplication:

define f \x x⋅x
5 + (f 3)⋅2 => 23

Note that * works for multiplication as well (and is the same thing):

define f \x x⋅x
5 + (f 3)*2 => 23

Reflection

Sometimes you want to know how a function was originally defined (as opposed to how it was actually implemented).

Therefore, the REPL has a command to retrieve the original definition:

Note that you need to put parenthesis around operators (in order to prevent them from asserting their own precedence).

define f \x x*x
describe 'f => \x x*x
describe '(;) => \m \ber \world (\r ber (ioValue r) (ioWorld r)) (m world)

Recursion

Usually, the function name is not available within the function:

define f \x (f 1)      -- makes no sense

If you want it to do recursion, do it like this:

define f rec \f \x (f 1)      -- endless recursion, but it works

Or like this, which is exactly the same:

define f rec \self \x (self 1)      -- endless recursion, but it works

There's a more traditional way, too:

defrec f \x (f 1)      -- endless recursion, but it works

defrec is a macro which expands to the former. Note that defrec only works (usefully) for function definitions.

Note that the reason that define f doesn't make f available within the body of f is that you can refer to previous definitions of f within the new definition of f.

An example will make that clearer:

define f \x (x**x)
define f \x if (x = 0)   1
                         (f x)    -- refers to the one above.

Lists

The simplest form of list is the linked list.

Nodes of a linked list are created by : like this:

42:24:12:nil

This is a list with 3 elements, linked together through the tail. The last element has nil tail.

head (1:2:3:nil)            => 1
tail (1:2:3:nil)            => (2:3:nil)
head (tail (1:2:3:nil))     => 2
cons? (1:nil)               => #t
cons? nil                   => #f
cons? 42                    => #f
list? (1:nil)               => #t
list? nil                   => #t

There's also a shorthand macro for creating lists used like this:

[42 24 12]

Literal Names

Sometimes you want to not reduce a function or operator to their definition but rather leave the name. To do so, prefix their name by quote or '

define x 2
x            => 2
quote x      => x       -- NOT 2
'x           => x       -- NOT 2
symbol? 'x   => #t
symbol? x    => #f

Conditionals

Conditionals are used for flow control. Note that the result of a conditional is a Church functional.

If you want to see a short result on screen, use toBool. toBool converts a Chruch Boolean to either the symbol t or the symbol f after figuring out which is appropriate.

#t&&#t => \t \f (\t \f t) ((\t \f t) t f) f
toBool (#t&&#t) => t
#t&&#t | toBool => t

I/O

runIO
         write stdout "What's your name? " ;\_
         flush stdout ;\_
         readline stdIn ;\name
         write stdout "Hello " ;\_
         write stdout name ;\_
         flush stdout ;\_
         return name

What this does is:

runIO wraps IO.

write \file \text will write text, once a World is given (i.e. by runIO or ;).

; will sequence two operations so the left-hand-side operation will be done first and the right-hand-side operation will be done afterwards.

_ is the traditional variable name for "don't care about the result".

Note that the parens have been left out and replaced by indentation.

Indentation

If you want to spread your programs over multiple lines, you can. However, then, indentation rules come into effect.

if (x = 0)
  1
$elif (x = 1)
  2
$elif (x = 2)
  3
$else 4

These work as follows:

If the indentation (number of whitespace characters at the beginning of the line) increased compared to the previous line, then a "(" is inserted.

If the indentation decreased compared to the previous lines, then one or more ")" are inserted until previous blocks up and excluding the current block are closed.
